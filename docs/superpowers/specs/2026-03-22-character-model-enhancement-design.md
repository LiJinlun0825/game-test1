# 角色建模优化设计规格

创建日期：2026-03-22

## 概述

优化游戏中的角色建模，通过程序化生成方式实现高度写实的几何细节和丰富的动画系统。

## 目标

- 几何细节：高面数人体模型，接近真实人体比例
- 动画系统：基础动作组 + 战斗动作组，支持动画混合

## 技术方案

采用增强型模型工厂方案，扩展现有 `HumanModelFactory`，增加骨骼层级系统和更细致的几何体。

## 几何细节设计

### 头部（约200面）
- 更真实的头骨形状（前后略扁的椭球）
- 眼窝凹陷 + 眉骨
- 耳朵（简化的耳廓形状）
- 鼻梁和鼻孔
- 嘴唇轮廓
- 下颌线条
- 可选的发型变体

### 躯干（约150面）
- 胸肌轮廓
- 腹部肌肉暗示
- 肩膀斜方肌
- 骨盆区域过渡
- 背部脊柱曲线暗示

### 四肢（约200面）
- 手臂：三角肌、二头肌/三头肌暗示、前臂肌肉曲线
- 手部：5根手指（各3节）、手掌、手腕
- 腿部：大腿四头肌、小腿肌肉曲线、膝盖髌骨
- 脚部：脚踝、脚掌、脚趾（简化）

### 比例参考（成年男性）
- 总身高：1.75米
- 头身比：7.5-8头身
- 肩宽：约0.45米

## 动画系统设计

### 基础动作组
| 动画 | 描述 | 过渡 |
|------|------|------|
| Idle | 待机呼吸动画，轻微身体起伏 | 默认状态 |
| Walk | 行走循环，脚跟着地节奏 | ←→ Run 平滑过渡 |
| Run | 奔跑循环，更大摆幅 | ←→ Walk 平滑过渡 |
| Jump | 起跳 → 空中 → 落地三阶段 | 触发式 |
| Crouch | 蹲下姿态，降低重心 | 平滑切换 |
| Stand | 从蹲下/其他姿态恢复站立 | 平滑切换 |

### 战斗动作组
| 动画 | 描述 | 触发条件 |
|------|------|----------|
| Aim | 举枪瞄准姿态，上半身朝向目标 | 按住瞄准键 |
| Shoot | 射击后坐力动画，手臂后移复位 | 开火时 |
| Reload | 换弹动作，双手操作武器 | 换弹时 |
| Hurt | 受伤反应，身体后仰/踉跄 | 受到伤害时 |
| Death | 倒地动画，身体瘫软倒下 | 生命值归零 |

### 动画混合策略
- **上半身/下半身分离**：下半身走/跑，上半身可独立瞄准/射击
- **动画层叠加**：呼吸动画作为基础层，动作动画叠加其上
- **过渡时间**：普通动作 0.2秒，战斗动作 0.1秒

## 技术架构

### 文件结构
```
src/model/
├── HumanModelFactory.js      # 增强版模型工厂
├── SkeletonBuilder.js        # 骨骼层级构建器
├── BodyPartBuilder.js        # 身体部位几何构建器
└── AnimationController.js    # 动画控制器

src/animation/
├── AnimationState.js         # 动画状态定义
├── AnimationBlender.js       # 动画混合器
└── AnimationLibrary.js       # 动画数据库（程序化生成的关键帧）
```

### 核心类设计

**HumanModelFactory**
- 创建完整骨骼层级（THREE.Bone 链）
- 生成各身体部位网格并绑定到骨骼
- 提供动画控制器接口

**AnimationController**
- 管理当前动画状态（状态机）
- 处理动画过渡和混合
- 支持上下半身独立动画

**AnimationLibrary**
- 存储所有动画的关键帧数据
- 提供动画查询和插值接口

### 数据流
```
用户输入 → AnimationController.update() →
AnimationBlender.interpolate() →
Bone.rotation 更新 → 渲染
```

## 实现计划

### 阶段1：骨骼系统基础
- 创建 SkeletonBuilder 构建完整骨骼层级
- 修改 HumanModelFactory 使用骨骼绑定网格

### 阶段2：几何细节增强
- 实现 BodyPartBuilder 创建高细节身体部位
- 更新模型工厂使用新几何体

### 阶段3：动画系统
- 实现 AnimationLibrary 程序化生成动画关键帧
- 实现 AnimationBlender 处理混合和过渡
- 实现 AnimationController 状态机

### 阶段4：集成与优化
- 更新 Player.js 和 Enemy.js 使用新动画系统
- 性能优化（LOD、减少更新频率）

## 约束

- 纯程序化生成，不依赖外部模型资源
- 保持与现有 Entity/Player/Enemy 架构兼容
- Web 环境性能优化（控制面数、合理使用 LOD）
